## ⚙️ Using the Tool

The software interface remains simple. Follow this workflow to start your first performance test.

### Launch the Application
Double-click the desktop icon. The main window opens. You see a dashboard that displays your current connection status to the MCP servers.

### Configure Servers
Click the Settings button. Enter the addresses for the MCP servers that you wish to compare. If you already have your credentials for these servers, enter them in the provided text boxes. Click Save after you enter the details.

### Select Your Code Base
Click the Select Folder button. Browse your computer to find the 1C project files that you want to analyze. The software scans the folder to ensure it contains compatible files.

### Start the Comparison
Press the Run Test button. The software begins the evaluation process. A progress bar shows you how much work remains. Do not close the window while the test runs.

### Review Results
Once the status bar reaches one hundred percent, the software generates a report. This document displays the response time and the logical accuracy of each server. You can export this report as a PDF by clicking the Export button.

## 🛠️ Troubleshooting Common Issues

Software occasionally behaves in unexpected ways. If you face a problem, use these steps to fix the issue.

*   Application does not open: Restart your computer and try to launch the program again.
*   Cannot find server: Check your internet connection. Ensure the server address you entered in the Settings menu is correct.
*   Slow performance: Close other large programs on your computer to free up memory for the comparison tool.
*   File errors: Make sure the 1C files you chose are not open in another editor or program during your test.

If the program still fails to work, uninstall the software and perform the download and installation steps again. Sometimes a clean reinstallation fixes small glitches.

## 📑 Understanding Your Data

The report you get at the end of the test contains specific metrics. Understanding these terms helps you make better choices for your projects.

Latency represents the delay between your request and the server response. A low latency score indicates a faster server. The tool measures this in milliseconds.

Accuracy scores denote how well the server understood the specific nature of the 1C code provided. Higher accuracy scores mean the server correctly identified patterns and syntax within your files. Use these scores to select the best server for your long-term code maintenance.
